21: # OmicsClaw Skill Guide — scRNA Quick Start
22: 
23: **Status**: beginner-oriented workflow guide for users who do not already know
24: which single-cell skill to run first.
25: 
26: ## Who This Is For
27: 
28: Use this guide when the user says something like:
29: 
30: - “我只有单细胞 FASTQ，下一步该做什么？”
31: - “我不懂单细胞流程，帮我一步一步跑”
32: - “我有 10x 数据，应该先 QC 还是先转 h5ad？”
33: 
34: This guide is intentionally opinionated and focuses on the current mainstream
35: OmicsClaw scRNA path.
36: 
37: Important beginner distinction:
38: 
39: - `sc-enrichment` = statistical GO / KEGG / Hallmark term enrichment on marker or DE rankings
40: - `sc-pathway-scoring` = per-cell pathway / signature activity scoring
41: 
42: ## The Default Route
43: 
44: For a normal 10x scRNA project, the recommended path is:
45: 
46: 1. `sc-fastq-qc`
47: 2. `sc-count`
48: 3. `sc-qc`
49: 4. `sc-preprocessing`
50: 5. optional branch:
51:    - `sc-doublet-detection` before interpretation if doublets are a concern
52: 6. optional branch:
53:    - `sc-batch-integration` if there are multiple batches / donors / libraries
54: 7. `sc-clustering`
55: 8. downstream analysis such as:
56:    - `sc-markers`
57:    - `sc-pseudotime` when the user already knows the likely start state and wants lineage ordering
58:    - `sc-cell-communication`
59:    - `sc-enrichment`
60:    - `sc-pathway-scoring`
61:    - `sc-cell-annotation`
62:    - `sc-de`
63:    - `sc-pseudotime`
64: 
65: If the user wants RNA velocity, insert:
66: 
67: 1. `sc-velocity-prep`
68: 2. `sc-velocity`
69: 
70: after preprocessing or after a suitable preprocessed object already exists.
71: 
72: ## Start Here: What Does The User Actually Have?
73: 
74: ### Case 1: The user has raw FASTQ files
75: 
76: This is the most common beginner starting point.
77: 
78: Recommended route:
79: 
80: 1. run `sc-fastq-qc`
81: 2. if quality is acceptable, run `sc-count`
82: 3. pass the resulting `processed.h5ad` into `sc-qc`
83: 4. then run `sc-preprocessing`
84: 5. if doublets matter, run `sc-doublet-detection`
85: 6. if multiple batches exist, run `sc-batch-integration`
86: 7. then run `sc-clustering`
87: 
88: If the user explicitly wants:
89: 
90: - **Cell Ranger or STARsolo**: use `sc-count`
91: - **SimpleAF / Alevin-fry or kb-python**: still use `sc-count`, but switch `--method`
92: 
93: ### Case 2: The user already has Cell Ranger or STARsolo output
94: 
95: Do **not** ask them to rerun FASTQ processing.
96: 
97: Recommended route:
98: 
99: 1. import with `sc-count`
100: 2. continue with `sc-qc`
101: 3. continue with `sc-preprocessing`
102: 4. then either `sc-batch-integration` or directly `sc-clustering`
103: 
104: ### Case 3: The user already has an external `.h5ad`
105: 
106: Recommended route:
107: 
108: 1. run `sc-standardize-input`
109: 2. then `sc-qc`
110: 3. then `sc-preprocessing`
111: 4. then either `sc-batch-integration` or directly `sc-clustering`
112: 
113: This is the safest way to avoid hidden count-vs-normalized matrix problems.
114: 
115: ### Case 4: The user wants RNA velocity
116: 
117: Ask whether they already have:
118: 
119: - a loom file
120: - a Cell Ranger BAM
121: - or STARsolo Velocyto output
122: 
123: Recommended route:
124: 
125: 1. `sc-velocity-prep`
126: 2. `sc-velocity`
127: 
128: Do **not** send users directly to `sc-velocity` if `spliced` and `unspliced`
129: layers do not already exist.
130: 
131: ## Which Counting Skill Should A Beginner Choose?
132: 
133: ### The safe default: `sc-count`
134: 
135: Use `sc-count` when:
136: 
137: - the user is on a mainstream 10x path
138: - they want Cell Ranger
139: - they want STARsolo
140: - they already have Cell Ranger or STARsolo outputs
141: 
142: ### Advanced counting methods inside `sc-count`
143: 
144: If a user explicitly wants pseudoalignment, keep them on `sc-count` and switch:
145: 
146: - `--method simpleaf`
147: - `--method kb_python`
148: 
149: For a beginner who says nothing special, `sc-count --method cellranger` is the
150: more natural first default.
151: 
152: ## Where Should A Beginner Put References?
153: 
154: If the user does not want to keep typing long absolute paths, recommend this
155: layout:
156: 
157: ```text
158: resources/singlecell/references/
159: ├── cellranger/
160: ├── starsolo/
161: ├── simpleaf/
162: ├── kb/
163: ├── whitelists/
164: └── gtf/
165: ```
166: 
167: Then explain the rule in simple words:
168: 
169: - if there is only one obvious reference in the right directory, OmicsClaw can auto-detect it
170: - if there are multiple versions, the user should pass `--reference`, `--t2g`, `--gtf`, or `--whitelist` explicitly
171: - Cell Ranger references come from 10x official reference releases
172: - STARsolo usually reuses the same FASTA / GTF plus a STAR-built genome directory
173: - `sc-velocity-prep --method velocyto` usually reuses the same `genes.gtf` as the counting reference
174: 
175: ## Environment Advice
176: 
177: For users who really want to run upstream locally, the safest recommendation is:
178: 
179: 1. keep a dedicated scRNA upstream environment
180: 2. install Python-side extras explicitly
181: 3. install large external tools manually
182: 
183: Recommended Python extras:
184: 
185: - `pip install -e ".[singlecell-upstream]"`
186: - `pip install -e ".[singlecell-velocity]"`
187: 
188: External tools that should be installed manually instead of auto-downloaded during a run:
189: 
190: - `fastqc`
191: - `multiqc`
192: - `STAR`
193: - `cellranger`
194: - `simpleaf`
195: - `kb`
196: - `velocyto`
197: 
198: Important rule:
199: 
200: - OmicsClaw should guide users to install these tools and place references locally, but should not silently mutate the user's environment during normal skill execution.
201: 
202: ## A Beginner-Friendly Command Sequence
203: 
204: ### Standard 10x FASTQ route
205: 
206: ```bash
207: oc run sc-fastq-qc --input fastqs/ --output output/sc_fastq_qc
208: oc run sc-count --input fastqs/ --method cellranger --reference /path/to/refdata-gex-GRCh38-2020-A --output output/sc_count
209: oc run sc-qc --input output/sc_count/processed.h5ad --output output/sc_qc
210: oc run sc-preprocessing --input output/sc_count/processed.h5ad --output output/sc_preprocessing
211: oc run sc-clustering --input output/sc_preprocessing/processed.h5ad --output output/sc_clustering
212: ```
213: 
214: ### Existing `.h5ad` route
215: 
216: ```bash
217: oc run sc-standardize-input --input data.h5ad --output output/sc_standardize
218: oc run sc-qc --input output/sc_standardize/processed.h5ad --output output/sc_qc
219: oc run sc-preprocessing --input output/sc_standardize/processed.h5ad --output output/sc_preprocessing
220: oc run sc-clustering --input output/sc_preprocessing/processed.h5ad --output output/sc_clustering
221: ```
222: 
223: ### Velocity route
224: 
225: ```bash
226: oc run sc-velocity-prep --input sample_count/ --method velocyto --gtf genes.gtf --output output/sc_velocity_prep
227: oc run sc-velocity --input output/sc_velocity_prep/velocity_input.h5ad --output output/sc_velocity
228: ```

230: ## The Main User Mistakes To Prevent
231: 
232: ### Mistake 1: Jumping straight from FASTQ to clustering
233: 
234: Correct route:
235: 
236: - FASTQ QC
237: - counting
238: - QC
239: - base preprocessing
240: - clustering
241: 
242: ### Mistake 2: Treating any `.h5ad` as ready for downstream analysis
243: 
244: Correct route:
245: 
246: - standardize first when provenance is unclear
247: 
248: ### Mistake 3: Running `sc-velocity` on an ordinary count matrix
249: 
250: Correct route:
251: 
252: - prepare velocity layers first with `sc-velocity-prep`
253: 
254: ## Simple Decision Rule For Routing
255: 
256: If the user only says:
257: 
258: - “10x FASTQ”
259: - “single-cell FASTQ”
260: - “raw single-cell data”
261: 
262: route them to:
263: 
264: - `sc-fastq-qc`, then `sc-count`
265: 
266: If they say:
267: 
268: - “Cell Ranger output”
269: - “STARsolo output”
270: 
271: route them to:
272: 
273: - `sc-count`
274: 
275: If they say:
276: 
277: - “Alevin-fry”
278: - “simpleaf”
279: - “kb-python”
280: 
281: route them to:
282: 
283: - `sc-count` with the corresponding `--method`
284: 
285: If they say:
286: 
287: - “velocity”
288: - “loom”
289: - “spliced/unspliced”
290: 
291: route them to:
292: 
293: - `sc-velocity-prep`
294: 
295: If they say:
296: 
297: - “CITE-seq”
298: - “ADT”
299: - “HTO”
300: - “cellranger multi”
301: 
302: then explain that this is outside the current mainline scRNA workflow and
303: should be handled in a future dedicated single-cell multi-omics path.
