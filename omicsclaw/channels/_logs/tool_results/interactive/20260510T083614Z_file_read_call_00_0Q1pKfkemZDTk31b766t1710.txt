36:     1) Research context & scope (from paper + idea)

43:        - EXACT QC/filtering parameters. You MUST extract filtering thresholds (min_genes, min_cells, max_mt_pct, etc.) from the paper. If the paper does not specify these, explicitly instruct the next agent to use DATA-DRIVEN thresholds (e.g., 3-5 MADs from median) or highly permissive defaults (e.g., min_genes=20) to avoid over-filtering. Do NOT use arbitrary strict defaults.

45:        - expected artifacts (tables/plots/h5ad outputs)

116:     Before writing ANY code, use `read_file` to read `plan.md` in the

125:     ═══════════════════════════════════════════════════════════════════
126:     STEP 0 — SEARCH FOR SKILLS (mandatory before writing any code)
127:     ═══════════════════════════════════════════════════════════════════
128:     Before writing code for an analysis step, ALWAYS call skill_search
129:     with a relevant keyword (e.g. "differential expression", "QC",

133:     ═══════════════════════════════════════════════════════════════════
134:     TIER 1 — USE SKILL FUNCTIONS via load_skill() (preferred)

136:     If skill_search returns a match, use the load_skill() helper (already
137:     available in the kernel) to load and call skill functions. This is the

139: 
140:     CORRECT PATTERN (one example — skill_search gives you the exact functions):
141:       mod = load_skill("spatial-preprocessing")

143: 
144:     IMPORTANT: Do NOT guess function names. Call skill_search first — it returns
145:     the exact function list (marked with ▶) for each skill. Use those names.
146: 
147:     NOTE: Always call skill_search FIRST to see the actual available functions.
148:     The example above shows the general pattern, but skill_search returns the
149:     up-to-date function list for each skill.
150: 
151:     DO NOT try to use `from skills.spatial... import ...` — skill directories
152:     use hyphens and have no __init__.py, so normal Python imports will FAIL.
153:     Always use load_skill("<skill-name>") which handles this correctly.
154: 
155:     CRITICAL QC PARAMETER RULE:
156:     When calling preprocessing/QC skills, you MUST strictly follow the filtering
157:     parameters defined in the plan.md. If the plan specifies "data-driven" or if
158:     you are unsure, use highly permissive thresholds (e.g., min_genes=20, min_cells=3,
159:     max_mt_pct=25) or calculate thresholds dynamically based on MAD. Do NOT use strict
160:     arbitrary defaults (like min_genes=200 or 500) that might filter out half the dataset!

165:     If skill_search returns "No skills found", you MAY write custom

177:     NOTEBOOK EXECUTION WORKFLOW
178:     ═══════════════════════════════════════════════════════════════════
179:     1. Call notebook_create with a descriptive filename (e.g. "analysis.ipynb"

195:     - If a method fails, try the lightweight fallback method before escalating.
196:     - Provide clear diagnostics: root cause, fix applied, verification steps.
