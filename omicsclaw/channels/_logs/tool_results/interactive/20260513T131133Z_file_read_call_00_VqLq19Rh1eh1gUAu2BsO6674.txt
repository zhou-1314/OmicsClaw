14: 11: # OmicsClaw Skill Guide — SC QC
15: 12: 
16: 13: **Status**: implementation-aligned guide derived from the current OmicsClaw
17: 14: `sc-qc` skill. This is a living wrapper guide for execution reasoning and
18: 15: output interpretation. It is not a claim that OmicsClaw already implements all
19: 16: possible single-cell QC frameworks.
20: 17: 
21: 18: ## Purpose
22: 19: 
23: 20: Use this guide when you need to decide:
24: 21: 
25: 22: - whether `sc-qc` is the right skill before filtering
26: 23: - how to explain the current `qc_metrics` implementation honestly
27: 24: - how to interpret the standard QC gallery before moving to `sc-preprocessing`
28: 25: 
29: 26: ## Step 1: Inspect The Data First
30: 27: 
31: 28: If the dataset has not been inspected yet in the conversation, inspect it
32: 29: before running QC.
33: 30: 
34: 31: Key properties to check:
35: 32: 
36: 33: - **Matrix type**:
37: 34:   - current `sc-qc` works best when a raw count-like matrix exists in `layers["counts"]`, aligned `adata.raw`, or `adata.X`
38: 35:   - if `adata.X` already looks log-normalized or scaled, say that explicitly and let the wrapper fall back to counts when available
39: 36: - **Input provenance**:
40: 37:   - if counts or gene symbols come from an external object with unclear provenance, recommend `sc-standardize-input` first
41: 38: - **Gene naming convention**:
42: 39:   - `human` expects mitochondrial `MT-` and ribosomal `RP[SL]`
43: 40:   - `mouse` expects mitochondrial `mt-` and ribosomal `Rp[sl]`
44: 41:   - Ensembl-only IDs or custom symbols can weaken `%MT` and `%ribo` estimates
45: 42: - **Dataset scale**:
46: 43:   - very sparse data tends to show broader low-count tails
47: 44:   - very large datasets may still be fine here because the skill only computes QC metrics and plots
48: 45: - **Biological context**:
49: 46:   - blood, tumor, immune, and metabolic tissues can imply different acceptable `%MT` ranges
50: 47: 
51: 48: Important implementation notes in current OmicsClaw:
52: 49: 
53: 50: - The current skill exposes one public method: `qc_metrics`.
54: 51: - `--species` is the only public extra flag.
55: 52: - Ribosomal percentage is always computed in the current wrapper.
56: 53: - The skill writes QC annotations into a canonical downstream-ready `processed.h5ad` but does not remove any cells.
57: 54: - The current wrapper now shares the same preflight style as other main scRNA skills: no raw count-like matrix should hard-stop the run; weaker gene-symbol matching should produce warnings rather than fake precision.
58: 55: - Loader, preflight, and standardization now have separate roles: `smart_load` reads and records a basic contract, `preflight` decides what to tell the user, and `sc-standardize-input` is the explicit repair path when provenance is messy.
59: 56: - The saved `processed.h5ad` is intentionally still a count-oriented object: `adata.X` is raw count-like, `adata.layers["counts"]` is the canonical raw layer, and `adata.raw` is a count-like snapshot. Normalized-expression workflows usually still need `sc-preprocessing` next.
60: 57: 
61: 58: ## Step 2: Explain The Current Skill Correctly
62: 59: 
63: 60: Current OmicsClaw `sc-qc` does:
64: 61: 
65: 62: 1. load data through the shared single-cell loader
66: 63: 2. detect mitochondrial and ribosomal genes from `--species`
67: 64: 3. calculate Scanpy QC metrics
68: 65: 4. add log-transformed helper metrics
69: 66: 5. render the standard OmicsClaw QC gallery through the shared single-cell `_lib/viz` layer
70: 67: 6. export tables, figure-data CSVs, report, result JSON, AnnData, README, and reproducibility files
71: 68: 
72: 69: Current OmicsClaw `sc-qc` does **not**:
73: 70: 
74: 71: 1. choose filtering cutoffs automatically
75: 72: 2. apply cell filtering
76: 73: 3. apply gene filtering
77: 74: 4. run doublet detection
78: 75: 5. normalize or cluster the data
79: 76: 
80: 77: Do not describe the skill as if it already performs full QC decision-making.
81: 78: It is a diagnostics and evidence-generation step.
82: 79: 
83: 80: ## Step 3: Handle The Only Public Tuning Knob Properly
84: 81: 
85: 82: ### `species`
86: 83: 
87: 84: Tune in this order:
88: 85: 
89: 86: 1. `species`
90: 87: 
91: 88: Guidance:
92: 89: 
93: 90: - Use `human` when gene symbols follow uppercase human conventions such as `MT-ND1` and `RPL13`.
94: 91: - Use `mouse` when gene symbols follow mixed-case mouse conventions such as `mt-Nd1` and `Rpl13`.
95: 92: - If the dataset uses Ensembl IDs or custom renamed features, tell the user that mitochondrial and ribosomal tagging may be incomplete before the run.
96: 93: 
97: 94: Important warning:
98: 95: 
99: 96: - `species` is a wrapper-level control, but it materially changes QC percentages by controlling which genes are tagged as mitochondrial or ribosomal.
100: 97: 
101: 98: ## Step 4: Show An Effective Run Summary Before Execution
102: 99: 
103: 100: Before execution, give a short explicit summary such as:
104: 101: 
105: 102: ```text
106: 103: About to run single-cell QC
107: 104:   Method: qc_metrics
108: 105:   Species: human
109: 106:   Ribosomal percentage: enabled by default in current wrapper
110: 107:   Scope: diagnostic only; no cells or genes will be filtered
111: 108: ```
112: 109: 
113: 110: This matters because users often hear "QC" and assume filtering is included.
114: 111: 
115: 112: ## Step 5: Interpret The Gallery In Context
116: 113: 
117: 114: ### `n_genes_by_counts`
118: 115: 
119: 116: - A low-complexity tail can suggest empty droplets or damaged cells.
120: 117: - A very high-complexity tail can suggest doublets or merged barcodes.
121: 118: - Do not reuse one universal cutoff across all tissues.
122: 119: 
123: 120: ### `total_counts`
124: 121: 
125: 122: - Very low counts may indicate weak capture or low RNA content.
126: 123: - Very high counts may suggest doublets or highly loaded droplets.
127: 124: - Always interpret together with `n_genes_by_counts`.
128: 125: 
129: 126: ### `pct_counts_mt`
130: 127: 
131: 128: - Elevated `%MT` often suggests stressed or dying cells.
132: 129: - PBMC-like data often supports stricter thresholds.
133: 130: - Solid tissues and tumors often need broader tolerance.
134: 131: - Metabolically active tissues may show biologically elevated `%MT`.
135: 132: 
136: 133: ### `pct_counts_ribo`
137: 134: 
138: 135: - High ribosomal fraction can reflect biology or technical dominance.
139: 136: - Use it as supporting evidence, not as a standalone exclusion rule.
140: 137: 
141: 138: ### Highest-expressed-gene panel
142: 139: 
143: 140: - Look for dominant contaminants or suspiciously overwhelming features.
144: 141: - This panel is especially useful when one or two genes consume a large fraction of the library.
145: 142: 
146: 143: ## Step 6: What To Say After The Run
147: 144: 
148: 145: - If `%MT` appears uniformly low but gene naming is nonstandard: warn that tagging may be incomplete.
149: 146: - If the low-count tail is large: suggest reviewing cell-quality thresholds in `sc-preprocessing`.
150: 147: - If the high-count/high-gene tail is obvious: suggest follow-up doublet analysis rather than immediately discarding cells blindly.
151: 148: - If distributions differ strongly across known samples or batches: note that later batch-aware QC reasoning may be needed.
152: 149: - Always restate that no filtering was applied by `sc-qc`.
153: 150: 
154: 151: ## Step 7: Explain Outputs Correctly
155: 152: 
156: 153: When summarizing results:
157: 154: 
158: 155: - describe `processed.h5ad` as the canonical downstream-ready AnnData with standardized scRNA contract fields plus QC annotations, `layers["counts"]`, and `adata.raw`
159: 156: - describe `figures/` as the standard Python gallery users should inspect first, including the barcode-rank curve and QC correlation heatmap
160: 157: - describe `figure_data/` as the stable hand-off layer for downstream custom plotting, including future R-side styling
161: 158: - describe `tables/qc_metrics_summary.csv` as the compact metric summary
162: 159: - describe `tables/qc_metrics_per_cell.csv` as the per-cell export for threshold review or custom plots
163: 160: - describe `result.json.data.params` as the replayable public CLI settings
164: 161: - describe `result.json.data.effective_params` as the actual runtime configuration, including fixed wrapper behavior
165: 162: - describe `result.json.data.visualization` as the structured gallery contract
166: 163: 
167: 164: Do **not** imply that the skill filtered cells successfully simply because it
168: 165: produced QC plots. The correct next step is to decide thresholds or pass the
169: 166: annotated object into a filtering/preprocessing skill.
