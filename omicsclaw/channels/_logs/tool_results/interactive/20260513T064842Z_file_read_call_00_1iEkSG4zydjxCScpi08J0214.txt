24: # spatial-preprocess
25: 
26: ## When to use
27: 
28: The user has a spatial AnnData (Visium / Xenium / SpaceRanger output /
29: generic) — either freshly loaded or coming out of
30: `spatial-raw-processing` — and wants the canonical
31: "QC → filter → normalise → HVG → PCA → neighbours → Leiden" path
32: producing a downstream-ready `processed.h5ad`. This is the **foundation
33: skill** — most other spatial analyses (`spatial-domains`,
34: `spatial-de`, `spatial-genes`, `spatial-deconv`,
35: `spatial-communication`, ...) consume its output. Single backend:
36: `scanpy_standard`.
37: 
38: For raw FASTQ → matrix conversion use `spatial-raw-processing`. For
39: explicit tissue-domain detection (SpaGCN / STAGATE) on top of this
40: output use `spatial-domains`.
41: 
42: ## Inputs & Outputs
43: 
44: | Input | Format | Required |
45: |---|---|---|
46: | Spatial AnnData | `.h5ad` (Visium / Xenium / generic, with `obsm["spatial"]` recommended) | yes (unless `--demo`) |
47: | Tissue preset | `--tissue` (one of `TISSUE_PRESETS`) | optional — overrides QC defaults |
48: 
49: | Output | Path | Notes |
50: |---|---|---|
51: | Processed AnnData | `processed.h5ad` | `X = normalised`, `layers["counts"] = raw`, `obsm["X_pca"]`, `obsm["X_umap"]`, `var["highly_variable"]`, `obs["leiden"]` |
52: | Run summary | `tables/preprocess_summary.csv` | always |
53: | Cluster sizes | `tables/cluster_summary.csv` | always |
54: | QC summary | `tables/qc_summary.csv` | per-cluster QC metrics |
55: | PCA variance | `tables/pca_variance_ratio.csv` | per-PC variance |
56: | Multi-resolution sweep | `tables/multi_resolution_summary.csv` | when `--resolutions` is provided |
57: | Report | `report.md` + `result.json` | always |

59: ## Flow
60: 
61: 1. Load AnnData (`--input`) or build a synthetic spatial demo.
62: 2. Apply tissue preset if `--tissue <preset>` is given (overrides default `--min-genes` / `--min-cells` / `--max-mt-pct` / `--max-genes`).
63: 3. QC + filter spots / genes; mitochondrial-percentage filter uses `--species` for gene prefix (`MT-` for human, `mt-` for mouse).
64: 4. Normalise (CP10k log) → HVG (`--n-top-hvg`) → PCA (`--n-pcs`).
65: 5. Build neighbour graph (`--n-neighbors`) → Leiden at `--leiden-resolution`.
66: 6. If `--resolutions a,b,c,...` is set, sweep additional Leiden resolutions and write the multi-resolution table.
67: 7. Save `processed.h5ad`, tables, figures, `report.md`, `result.json`.

69: ## Gotchas
70: 
71: - **All input + parameter validation goes through `parser.error` (exit code 2), not `ValueError` / `SystemExit(1)`.** `spatial_preprocess.py:1006` for missing `--input`; `:1008` for missing path; `:1010` for unknown `--tissue`; `:1012-1028` for negative / out-of-range numeric flags. Wrappers expecting standard `ValueError` need to handle exit code 2 separately.
72: - **`result.json["n_pcs_used"]` may be smaller than the requested `--n-pcs`.** The dimensionality is clipped in the helper at `skills/spatial/_lib/preprocessing.py:307` (cross-file anchor — lint skips). The clipped value is surfaced into the per-row metrics CSV at `spatial_preprocess.py:193` (`n_pcs_used`) and `:191` (`n_pcs_requested`). Pass `n_pcs_used` (not `n_pcs_requested`) to downstream skills like `spatial-domains --n-pcs`.
73: - **`--tissue` overrides numeric defaults silently.** When a preset matches, `TISSUE_PRESETS` rewrites `--min-genes` / `--min-cells` / `--max-mt-pct` / `--max-genes` from the preset table. Pass values explicitly to override; `result.json["effective_params"]` records what was actually applied.
74: - **No UMAP fallback — missing `umap-learn` aborts the run.** `sc.tl.umap` is called unconditionally; if `umap-learn` (or igraph for Leiden) isn't installed, the run raises `ImportError` and exits non-zero. There is no `_safe_umap` shim in the script — install `umap-learn` and `igraph` before running on a fresh env.
75: - **`--resolutions` parsing errors via `parser.error`.** `spatial_preprocess.py:1026` raises on malformed comma-separated values; `:1028` raises if any value is `<= 0`. Format: `0.4,0.6,0.8,1.0` (no spaces inside the value).
76: - **Default `--tissue` is `None` (no preset).** When unset, the skill uses the generic defaults from `defaults` dict at `:956-958` — typically the right call. Tissue presets (`brain`, `tumor`, etc.) tighten thresholds and may filter aggressively on tissues with low UMI counts.

78: ## Key CLI
79: 
80: ```bash
81: # Demo (synthetic Visium)
82: python omicsclaw.py run spatial-preprocess --demo --output /tmp/spatial_pp_demo
83: 
84: # Visium with default presets
85: python omicsclaw.py run spatial-preprocess \
86:   --input visium.h5ad --output results/
87: 
88: # Tissue preset + custom resolution
89: python omicsclaw.py run spatial-preprocess \
90:   --input visium.h5ad --output results/ \
91:   --data-type visium --tissue brain --leiden-resolution 1.2
92: 
93: # Multi-resolution sweep for picking optimal clustering
94: python omicsclaw.py run spatial-preprocess \
95:   --input visium.h5ad --output results/ \
96:   --resolutions 0.4,0.6,0.8,1.0,1.4
97: 
98: # Mouse Xenium
99: python omicsclaw.py run spatial-preprocess \
100:   --input xenium.h5ad --output results/ \
101:   --data-type xenium --species mouse --max-mt-pct 15
102: ```
