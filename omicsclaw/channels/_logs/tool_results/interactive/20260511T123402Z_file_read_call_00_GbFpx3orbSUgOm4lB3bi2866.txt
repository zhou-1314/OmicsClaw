10: # Single-Cell RNA-seq Core Analysis (Scanpy)
11: 
12: Complete workflow for single-cell RNA-seq analysis using Scanpy and the scverse ecosystem. Process raw data through quality control, normalization, clustering, and cell type annotation with publication-ready visualizations.
13: 
14: ## When to Use This Skill
15: 
16: - **Analyze 10X Chromium data** (CellRanger output, H5 files, raw/filtered matrices)
17: - **Process Drop-seq, Smart-seq2, or inDrop** single-cell RNA-seq data
18: - **Integrate multi-batch data** using scVI, scANVI, or Harmony
19: - **Annotate cell types** manually or with automated reference-based methods
20: - **Compare conditions** using pseudobulk differential expression (multi-sample data)
21: 
22: **Don't use for:** Bulk RNA-seq (use bulk-rnaseq-counts-to-de-deseq2), R-based scRNA-seq (use scrnaseq-seurat-core-analysis), Spatial transcriptomics (coming soon)
23:

45: ## Inputs
46: 
47: **Required:**
48: - **Raw or filtered count matrix:** CellRanger output (`filtered_feature_bc_matrix/`), H5 files (`.h5`), AnnData (`.h5ad`), or count matrices (CSV/TSV)
49: 
50: **Optional:** Sample metadata (CSV/TSV) with sample IDs, conditions, batches, donor IDs
51: 
52: **Data requirements:** Min 500 cells/sample (1000+ recommended), Human or Mouse, UMI-based or read counts. See [references/qc_guidelines.md](references/qc_guidelines.md) for tissue-specific thresholds.
53: 
54: ## Outputs
55: 
56: **Analysis objects:**
57: - `adata_processed.h5ad` - Complete annotated AnnData object
58:   - **Load with:** `import scanpy as sc; adata = sc.read_h5ad('adata_processed.h5ad')`
59:   - Contains: raw counts, normalized data, QC metrics, clusters, cell types, UMAP/PCA
60:   - **Note:** `adata.X` contains log-normalized (not scaled) data. Scaled data is used internally for PCA but not stored in .X. This is correct for downstream analysis (DE, visualization).
61:   - **Required for:** trajectory inference, cell-cell communication, downstream analyses
62: 
63: **Reports:**
64: - `scrna_analysis_report.pdf` - Agent-generated comprehensive PDF with Methods, Results, Figures, Conclusions
65: - `analysis_summary.txt` - Text summary of dataset, QC, clustering, integration (generated by `export_anndata_results()`)
66: 
67: **⚠️ PDF style rules:**
68: - **US Letter page size (8.5 × 11 in)** — always set page dimensions explicitly; do not rely on library defaults
69: - **No Unicode superscripts** — use `3.36e-06` or `3.36 × 10^(-6)`, not Unicode superscript chars (they render as ■ in PDF fonts)
70: - **No half-empty pages** — group headings with their content; only page-break before major sections (Results, Conclusions)
71: - **Figures ≥80% page width** — multi-panel figures must be large enough to read; never embed below 50% width
72: 
73: **Tables:** `cell_metadata.csv`, `expression_matrix_counts.csv`, `expression_matrix_normalized.csv`, `pca_coordinates.csv`, `umap_coordinates.csv`, `cluster_markers_all.csv`, `{celltype}_deseq2_results.csv`
74: 
75: **Visualizations (PNG + SVG at 300 DPI):** QC violins, UMAP plots, marker heatmaps, dot plots, volcano/MA plots

93:    - a) Single sample (no integration needed)

97:    - a) Standard: QC + normalize + cluster + annotate (recommended)
98:    - b) Standard + pseudobulk DE (requires ≥2 samples per condition)
99:    - c) Custom: specify which steps

101: ### 5. **Clustering granularity:**
102:    - a) Coarse (0.3-0.5) — major cell types only
103:    - b) Standard (0.6-0.8) — recommended
104:    - c) Fine (1.0-1.5) — subtypes
105:    - d) Test multiple: 0.4, 0.6, 0.8, 1.0 (recommended)
