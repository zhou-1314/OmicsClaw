11: # OmicsClaw Skill Guide — SC QC
12: 
13: **Status**: implementation-aligned guide derived from the current OmicsClaw
14: `sc-qc` skill. This is a living wrapper guide for execution reasoning and
15: output interpretation. It is not a claim that OmicsClaw already implements all
16: possible single-cell QC frameworks.
17: 
18: ## Purpose
19: 
20: Use this guide when you need to decide:
21: 
22: - whether `sc-qc` is the right skill before filtering
23: - how to explain the current `qc_metrics` implementation honestly
24: - how to interpret the standard QC gallery before moving to `sc-preprocessing`
25: 
26: ## Step 1: Inspect The Data First
27: 
28: If the dataset has not been inspected yet in the conversation, inspect it
29: before running QC.
30: 
31: Key properties to check:
32: 
33: - **Matrix type**:
34:   - current `sc-qc` works best when a raw count-like matrix exists in `layers["counts"]`, aligned `adata.raw`, or `adata.X`
35:   - if `adata.X` already looks log-normalized or scaled, say that explicitly and let the wrapper fall back to counts when available
36: - **Input provenance**:
37:   - if counts or gene symbols come from an external object with unclear provenance, recommend `sc-standardize-input` first
38: - **Gene naming convention**:
39:   - `human` expects mitochondrial `MT-` and ribosomal `RP[SL]`
40:   - `mouse` expects mitochondrial `mt-` and ribosomal `Rp[sl]`
41:   - Ensembl-only IDs or custom symbols can weaken `%MT` and `%ribo` estimates
42: - **Dataset scale**:
43:   - very sparse data tends to show broader low-count tails
44:   - very large datasets may still be fine here because the skill only computes QC metrics and plots
45: - **Biological context**:
46:   - blood, tumor, immune, and metabolic tissues can imply different acceptable `%MT` ranges
47: 
48: Important implementation notes in current OmicsClaw:
49: 
50: - The current skill exposes one public method: `qc_metrics`.
51: - `--species` is the only public extra flag.
52: - Ribosomal percentage is always computed in the current wrapper.
53: - The skill writes QC annotations into a canonical downstream-ready `processed.h5ad` but does not remove any cells.
54: - The current wrapper now shares the same preflight style as other main scRNA skills: no raw count-like matrix should hard-stop the run; weaker gene-symbol matching should produce warnings rather than fake precision.
55: - Loader, preflight, and standardization now have separate roles: `smart_load` reads and records a basic contract, `preflight` decides what to tell the user, and `sc-standardize-input` is the explicit repair path when provenance is messy.
56: - The saved `processed.h5ad` is intentionally still a count-oriented object: `adata.X` is raw count-like, `adata.layers["counts"]` is the canonical raw layer, and `adata.raw` is a count-like snapshot. Normalized-expression workflows usually still need `sc-preprocessing` next.
57: 
58: ## Step 2: Explain The Current Skill Correctly
59: 
60: Current OmicsClaw `sc-qc` does:
61: 
62: 1. load data through the shared single-cell loader
63: 2. detect mitochondrial and ribosomal genes from `--species`
64: 3. calculate Scanpy QC metrics
65: 4. add log-transformed helper metrics
66: 5. render the standard OmicsClaw QC gallery through the shared single-cell `_lib/viz` layer
67: 6. export tables, figure-data CSVs, report, result JSON, AnnData, README, and reproducibility files
68: 
69: Current OmicsClaw `sc-qc` does **not**:
70: 
71: 1. choose filtering cutoffs automatically
72: 2. apply cell filtering
73: 3. apply gene filtering
74: 4. run doublet detection
75: 5. normalize or cluster the data
76: 
77: Do not describe the skill as if it already performs full QC decision-making.
78: It is a diagnostics and evidence-generation step.
79: 
80: ## Step 3: Handle The Only Public Tuning Knob Properly
81: 
82: ### `species`
83: 
84: Tune in this order:
85: 
86: 1. `species`
87: 
88: Guidance:
89: 
90: - Use `human` when gene symbols follow uppercase human conventions such as `MT-ND1` and `RPL13`.
91: - Use `mouse` when gene symbols follow mixed-case mouse conventions such as `mt-Nd1` and `Rpl13`.
92: - If the dataset uses Ensembl IDs or custom renamed features, tell the user that mitochondrial and ribosomal tagging may be incomplete before the run.
93: 
94: Important warning:
95: 
96: - `species` is a wrapper-level control, but it materially changes QC percentages by controlling which genes are tagged as mitochondrial or ribosomal.
97: 
98: ## Step 4: Show An Effective Run Summary Before Execution
99: 
100: Before execution, give a short explicit summary such as:
101: 
102: ```text
103: About to run single-cell QC
104:   Method: qc_metrics
105:   Species: human
106:   Ribosomal percentage: enabled by default in current wrapper
107:   Scope: diagnostic only; no cells or genes will be filtered
108: ```
109: 
110: This matters because users often hear "QC" and assume filtering is included.
111: 
112: ## Step 5: Interpret The Gallery In Context
113: 
114: ### `n_genes_by_counts`
115: 
116: - A low-complexity tail can suggest empty droplets or damaged cells.
117: - A very high-complexity tail can suggest doublets or merged barcodes.
118: - Do not reuse one universal cutoff across all tissues.
119: 
120: ### `total_counts`
121: 
122: - Very low counts may indicate weak capture or low RNA content.
123: - Very high counts may suggest doublets or highly loaded droplets.
124: - Always interpret together with `n_genes_by_counts`.
125: 
126: ### `pct_counts_mt`
127: 
128: - Elevated `%MT` often suggests stressed or dying cells.
129: - PBMC-like data often supports stricter thresholds.
130: - Solid tissues and tumors often need broader tolerance.
131: - Metabolically active tissues may show biologically elevated `%MT`.
132: 
133: ### `pct_counts_ribo`
134: 
135: - High ribosomal fraction can reflect biology or technical dominance.
136: - Use it as supporting evidence, not as a standalone exclusion rule.
137: 
138: ### Highest-expressed-gene panel
139: 
140: - Look for dominant contaminants or suspiciously overwhelming features.
141: - This panel is especially useful when one or two genes consume a large fraction of the library.
142: 
143: ## Step 6: What To Say After The Run
144: 
145: - If `%MT` appears uniformly low but gene naming is nonstandard: warn that tagging may be incomplete.
146: - If the low-count tail is large: suggest reviewing cell-quality thresholds in `sc-preprocessing`.
147: - If the high-count/high-gene tail is obvious: suggest follow-up doublet analysis rather than immediately discarding cells blindly.
148: - If distributions differ strongly across known samples or batches: note that later batch-aware QC reasoning may be needed.
149: - Always restate that no filtering was applied by `sc-qc`.
150: 
151: ## Step 7: Explain Outputs Correctly
152: 
153: When summarizing results:
154: 
155: - describe `processed.h5ad` as the canonical downstream-ready AnnData with standardized scRNA contract fields plus QC annotations, `layers["counts"]`, and `adata.raw`
156: - describe `figures/` as the standard Python gallery users should inspect first, including the barcode-rank curve and QC correlation heatmap
157: - describe `figure_data/` as the stable hand-off layer for downstream custom plotting, including future R-side styling
158: - describe `tables/qc_metrics_summary.csv` as the compact metric summary
159: - describe `tables/qc_metrics_per_cell.csv` as the per-cell export for threshold review or custom plots
160: - describe `result.json.data.params` as the replayable public CLI settings
161: - describe `result.json.data.effective_params` as the actual runtime configuration, including fixed wrapper behavior
162: - describe `result.json.data.visualization` as the structured gallery contract
163: 
164: Do **not** imply that the skill filtered cells successfully simply because it
165: produced QC plots. The correct next step is to decide thresholds or pass the
166: annotated object into a filtering/preprocessing skill.
