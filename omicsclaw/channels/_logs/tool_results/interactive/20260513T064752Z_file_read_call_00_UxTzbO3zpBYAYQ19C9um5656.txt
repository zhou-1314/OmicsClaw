17: 14: 11: # OmicsClaw Skill Guide — Spatial Preprocess
18: 15: 12: 
19: 16: 13: **Status**: implementation-aligned guide derived from the current OmicsClaw
20: 17: 14: `spatial-preprocess` skill. This is **not** one of the 28 already validated
21: 18: 15: end-to-end workflows. It is a living guide for preprocessing parameter
22: 19: 16: reasoning, loader expectations, and wrapper-specific caveats.
23: 20: 17: 
24: 21: 18: ## Purpose
25: 22: 19: 
26: 23: 20: Use this guide when you need to decide:
27: 24: 21: - how to explain the current `scanpy_standard` preprocessing workflow
28: 25: 22: - which QC and graph parameters matter first for a spatial dataset
29: 26: 23: - how to distinguish wrapper-level loader controls from actual preprocessing controls
30: 27: 24: 
31: 28: 25: ## Step 1: Inspect The Data First
32: 29: 26: 
33: 30: 27: If the dataset has not been inspected yet in this conversation, call
34: 31: 28: `inspect_data` first.
35: 32: 29: 
36: 33: 30: Key properties to check:
37: 34: 31: - **Input type**:
38: 35: 32:   - `raw_counts.h5ad` emitted by `spatial-raw-processing`
39: 36: 33:   - Visium directory or feature matrix
40: 37: 34:   - Xenium `.zarr` / `.h5ad`
41: 38: 35:   - converted `.h5ad` for MERFISH / Slide-seq / seqFISH
42: 39: 36:   - if the user still has FASTQ pairs plus IDs / STAR reference files, route to `spatial-raw-processing` instead
43: 40: 37: - **Matrix representation**:
44: 41: 38:   - current wrapper expects raw counts in `X` on input
45: 42: 39:   - if the input already looks log-normalized or scaled, say that explicitly before preprocessing
46: 43: 40: - **Spatial coordinates**:
47: 44: 41:   - coordinates are strongly preferred
48: 45: 42:   - preprocessing can still run without them, but downstream spatial plots and analyses will be limited
49: 46: 43: - **Species**:
50: 47: 44:   - current wrapper only exposes `human` and `mouse`
51: 48: 45:   - this matters for mitochondrial prefix detection (`MT-` vs `mt-`)
52: 49: 46: - **Dataset size and sparsity**:
53: 50: 47:   - very sparse datasets interact strongly with `min_genes`, `max_mt_pct`, and `n_top_hvg`
54: 51: 48: - **Biological context**:
55: 52: 49:   - if the tissue is obvious, a tissue preset may be a reasonable first-pass shortcut
56: 53: 50: 
57: 54: 51: Important implementation notes in current OmicsClaw:
58: 55: 52: - The current skill exposes one implemented workflow: `scanpy_standard`.
59: 56: 53: - `data_type`, `species`, and `tissue` are wrapper-level controls, not alternative preprocessing algorithms.
60: 57: 54: - Tissue presets only fill thresholds when the user has not already supplied explicit values.
61: 58: 55: - Reports now record the **effective** thresholds after preset application.
62: 59: 56: - Raw counts are preserved in both `adata.layers["counts"]` and `adata.raw`.
63: 60: 57: - PCA is computed on the HVG subset, and the neighbor graph uses a clipped PC count that may be lower than the requested count if the data shape requires it.
64: 61: 58: 
65: 62: 59: ## Step 2: Explain The Current Workflow Correctly
66: 63: 60: 
67: 64: 61: Current OmicsClaw `spatial-preprocess` runs:
68: 65: 62: 1. QC metric calculation with mitochondrial annotation.
69: 66: 63: 2. Spot / gene filtering.
70: 67: 64: 3. Raw-count preservation.
71: 68: 65: 4. Library-size normalization and `log1p`.
72: 69: 66: 5. `seurat_v3` highly variable gene selection on the counts layer.
73: 70: 67: 6. Scaling, PCA, neighbors, UMAP, and Leiden clustering.
74: 71: 68: 
75: 72: 69: Do **not** describe this as if the user were choosing among multiple
76: 73: 70: implemented preprocessing backends. Right now, the choice is mainly about
77: 74: 71: thresholds, graph construction, and loader hints.
78: 75: 72: 
79: 76: 73: ## Step 3: Pick Parameters Deliberately
80: 77: 74: 
81: 78: 75: Use this quick guide when the user has not specified exact thresholds:
82: 79: 76: 
83: 80: 77: | Parameter group | Best first use | Strong starting point | Main caveat |
84: 81: 78: |-----------------|----------------|-----------------------|-------------|
85: 82: 79: | **Tissue preset** | Fast first pass when tissue identity is obvious | `tissue=brain`, `tissue=tumor`, etc. | Presets are wrapper-level shortcuts, not universal truth |
86: 83: 80: | **QC thresholds** | Core quality filtering | start with preset or `min_genes=200`, `max_mt_pct=10~20` depending on tissue | Over-filtering can remove biologically real regions |
87: 84: 81: | **HVG budget** | Feature selection for PCA / graph | `n_top_hvg=2000` | Too low can oversimplify; too high can add noise |
88: 85: 82: | **Graph size** | neighborhood structure | `n_pcs=30`, `n_neighbors=15` | Optimal values depend on dataset size and heterogeneity |
89: 86: 83: | **Leiden resolution** | cluster granularity | `leiden_resolution=0.5` | Higher resolution is not automatically better |
90: 87: 84: | **Resolution sweep** | exploratory clustering | `resolutions=0.4,0.6,0.8,1.0` | Useful for exploration, not always needed in the first pass |
91: 88: 85: 
92: 89: 86: ## Step 4: Always Show An Effective Parameter Summary Before Running
93: 90: 87: 
94: 91: 88: Before execution, tell the user what will be run in a short, concrete block:
95: 92: 89: 
96: 93: 90: ```text
97: 94: 91: About to run spatial preprocessing
98: 95: 92:   Workflow: scanpy_standard
99: 96: 93:   Loader: data_type=visium
100: 97: 94:   Effective QC: min_genes=200, min_cells=0, max_mt_pct=10, max_genes=6000
101: 98: 95:   Graph: n_top_hvg=2000, n_pcs=30, n_neighbors=15, leiden_resolution=0.5
102: 99: 96:   Note: brain preset was used to fill QC defaults, but these effective thresholds are what will actually be applied.
103: 100: 97: ```
104: 101: 98: 
105: 102: 99: This is important because saying only "I will use the brain preset" is not
106: 103: 100: precise enough for reproducibility.
107: 104: 101: 
108: 105: 102: ## Step 5: Parameter-Specific Tuning Rules
109: 106: 103: 
110: 107: 104: ### Tissue preset
111: 108: 105: 
112: 109: 106: Tune in this order:
113: 110: 107: 1. `tissue`
114: 111: 108: 2. explicit threshold overrides
115: 112: 109: 
116: 113: 110: Guidance:
117: 114: 111: - Use a tissue preset only when the tissue identity is reasonably clear.
118: 115: 112: - If the user already knows the right QC regime, explicit numeric thresholds are better than relying entirely on a preset.
119: 116: 113: - If a preset is used, explain the resolved values before execution.
120: 117: 114: 
121: 118: 115: Important warning:
122: 119: 116: - Tissue presets are OmicsClaw wrapper knowledge, not upstream Scanpy APIs.
123: 120: 117: 
124: 121: 118: ### QC thresholds
125: 122: 119: 
126: 123: 120: Tune in this order:
127: 124: 121: 1. `min_genes`
128: 125: 122: 2. `max_mt_pct`
129: 126: 123: 3. `max_genes`
130: 127: 124: 4. `min_cells`
131: 128: 125: 
132: 129: 126: Guidance:
133: 130: 127: - Raise `min_genes` when extremely sparse spots are dominating the dataset.
134: 131: 128: - Lower `min_genes` when sparse biology is expected and the user accepts more noise.
135: 132: 129: - Lower `max_mt_pct` when mitochondrial-rich low-quality spots are obvious.
136: 133: 130: - Raise `max_mt_pct` cautiously in tissues like heart or muscle where higher mitochondrial load may be biological.
137: 134: 131: - Use `max_genes` as an upper-bound guard when obvious doublet-like or overly complex spots are a concern.
138: 135: 132: - Use `min_cells` when you want to remove genes detected in very few spots before downstream modeling.
139: 136: 133: 
140: 137: 134: Important warning:
141: 138: 135: - Aggressive QC may remove real tissue niches, especially in sparse or damaged specimens.
142: 139: 136: 
143: 140: 137: ### HVG budget
144: 141: 138: 
145: 142: 139: Tune in this order:
146: 143: 140: 1. `n_top_hvg`
147: 144: 141: 
148: 145: 142: Guidance:
149: 146: 143: - Start with `n_top_hvg=2000`.
150: 147: 144: - Increase it when the dataset is large and biologically diverse.
