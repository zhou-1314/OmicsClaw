## 为什么用 skill 这种组织方式

**可发现性**：skill 即文件夹加 SKILL.md，`omicsclaw.py list` 一键全量列出所有已注册 skill，无需手工维护清单。新增一个目录并写好 SKILL.md，即可完成注册。

**可替换性**：每个 skill 自带 SKILL.md 描述输入/输出契约（`requires`、`saves_h5ad` 等字段），替换内部实现不影响调用方——orchestrator 和 bot 只依赖契约，不依赖实现细节。

**可教性**：SKILL.md 同时是文档和路由元数据，单一来源，LLM 编排器与人类用户共享同一份事实。前者从 frontmatter 读取 `trigger_keywords` 做路由打分，后者阅读 markdown body 获得方法论说明。


## 技能契约：skill.yaml + 生成的 SKILL.md

ADR 0037 起，每个 skill 的机器可读契约集中在一份 `skill.yaml`，由 `omicsclaw/skill/schema.py:SkillManifest`（pydantic）校验；registry 和 bot 都通过双轨的 `LazySkillMetadata` 从中读取字段（优先 `skill.yaml`，旧的 SKILL.md frontmatter 格式仅作安全兜底）。`SKILL.md` 不再手写元数据，而是由 `scripts/generate_skill_md.py` 从 `skill.yaml` **一键生成**——其 frontmatter 头部与 `## Inputs & Outputs` 段自动填充（不要手改），其余方法论正文（`When to use` / `Flow` / `Gotchas` / `Key CLI` / `See also`）人写并原样保留。

以下字段来自 `skills/spatial/spatial-preprocess/skill.yaml`：

| 字段 | 用途 | 示例（来自 spatial-preprocess） |
|---|---|---|
| `id` / `name` | skill 唯一标识（路由 slug） | `spatial-preprocess` |
| `domain` | 所属领域键（当前 8 个 domain） | `spatial` |
| `version` | 版本追踪 | `0.6.0` |
| `summary.load_when` | 一句话适用场景，供 LLM 路由打分 | `"running the foundational spatial ... QC + filtering + normalisation + HVG + PCA + ... + Leiden pipeline ..."` |
| `summary.skip_when` | 何时改用别的 skill（≥1 条，lint 强制） | `raw FASTQ → spatial-raw-processing`；`已预处理数据做域识别 → spatial-domains` |
| `summary.trigger_keywords` | 路由关键词打分 | `[preprocess, spatial QC, normalize, visium, xenium, leiden, umap, ...]` |
| `summary.tags` | 关键词列表，辅助搜索 | `[spatial, visium, xenium, preprocessing, qc, normalization, hvg, pca, leiden]` |
| `summary.aliases` | 旧名称，路由仍可命中 | `[preprocess, spatial-preprocessing]` |
| `interface.inputs` | 输入模态 / 文件类型 / 前置条件 | `modalities: [visium, xenium]`；`file_types: [h5ad]` |
| `interface.parameters.allowed_extra_flags` | bot 可传入的 CLI flags（安全白名单） | `["--min-genes", "--n-top-hvg", "--leiden-resolution", ...]` |
| `interface.parameters.hints` | autoagent 调参搜索空间 + bot 参数预览 | `scanpy_standard: {priority: "tissue → min_genes/max_mt_pct/... → leiden_resolution"}` |
| `interface.outputs.anndata.saves_h5ad` | 告知 LLM 该 skill 输出 h5ad | `true` |
| `interface.inputs.preconditions.data_shape.requires_preprocessed` | 告知 LLM 该 skill 是否需要预处理输入 | `false` |
| `runtime` | 运行语言与入口脚本 | `language: python`；`entry: spatial_preprocess.py` |
| `deps.python` | 第三方依赖（PyPI 名，自动装配） | `[anndata, matplotlib, numpy, pandas, scanpy, scipy, seaborn]` |

## Registry 加载流程

```
┌──────────────────────────────────────────────────────────┐
│  OmicsRegistry.load_all() 被调用                         │
└──────────────────────────────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│  _iter_skill_dirs() 遍历 skills/ 目录树                  │
│  · 平铺布局：skills/<domain>/<slug>/                     │
│  · 嵌套布局：skills/singlecell/scrna/<slug>/             │
│  · 含 SKILL.md 或单一 .py 文件的目录即为 skill           │
└──────────────────────────────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│  对每个 skill 目录：                                     │
│  · LazySkillMetadata 懒解析 skill.yaml（旧 frontmatter 兜底）│
│  · _resolve_script_path() 定位入口 .py 脚本              │
│  · 提取 domain / alias / legacy_aliases / demo_args      │
└──────────────────────────────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│  _register_skill_entry() 写入 registry.skills 字典       │
│  · canonical alias → info dict                           │
│  · legacy alias → 同一 info dict（多键同值）             │
└──────────────────────────────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│  _refresh_domain_skill_counts() 按实际数量更新域元数据   │
└──────────────────────────────────────────────────────────┘
                          ▼
┌──────────────────────────────────────────────────────────┐
│  orchestrator + run_skill 通过 registry.skills 查表分发  │
└──────────────────────────────────────────────────────────┘
```

## 启用前 / 启用后

| 维度 | 普通脚本集合 | OmicsClaw skill 体系 |
|---|---|---|
| 可发现性 | 翻 README 手找 | `omicsclaw.py list` 一行全出 |
| 可路由性 | 用户记住 CLI 名称 | orchestrator 自动打分选 |
| 可替换性 | 改名要全局搜替 | 只改自己目录内实现 |
| 可教性 | 文档与代码两边维护 | SKILL.md 单一来源 |

## 故障排查

| 症状 | 原因 | 解决 |
|---|---|---|
| `list` 报某 skill 不存在 | 该 skill 目录缺 SKILL.md | 补写 SKILL.md 后重新 list |
| SKILL.md frontmatter 解析报错 | YAML 语法错（缩进/引号） | 用 `python -c "import yaml; yaml.safe_load(open('skills/.../SKILL.md').read().split('---')[1])"` 调试 |
| skill 名冲突或路由跳错 | 两个 SKILL.md 的 name 字段相同 | 修改其中一个 name 字段使其唯一 |
