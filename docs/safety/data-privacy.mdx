## 为什么这一页很重要

基因组数据、单细胞测序结果、临床代谢组数据均涉及个人遗传特征，属于最高级别的敏感生物医学信息，一旦外泄后果不可逆。

即使工具声称"本地处理"，用户仍需要知道数据具体走哪条路径——不同部署模式下的边界差异显著，细节决定安全性。

LLM 调用是最容易被忽视的泄漏点：分析脚本确实在本地跑，但意图解析的文本提示（含文件路径、样本名等）会按照 OpenAI 兼容协议发往你配置的 LLM 端点。

## 三种部署模式的数据路径

### 模式 1：纯本地 CLI

直接调用 `omicsclaw.py run` 或各 skill 脚本时，所有计算均在本机 Python 进程内完成，不发起任何网络请求。

```text
你的数据（.h5ad / .csv / .fastq.gz）
   │
   ▼
omicsclaw.py run <skill> → 本地 skill 脚本 → 本地输出目录
   │
   ▼
无网络调用，数据完全不离开本机
```

纯 CLI 模式下，原始数据、中间矩阵与最终报告全部保留在用户指定的本地目录，不经过任何外部服务。

### 模式 2：Bot（Telegram / Feishu）

Bot 通道引入了两条额外路径，需要明确区分。

```text
用户文字消息
   │
   ├─→ Bot 平台服务器（Telegram / Feishu 云端）→ 本地 bot 进程
   │         ↑ 注意：Bot 平台缓存用户消息与上传图片
   │
   └─→ LLM 端点（由 .env 中 LLM_BASE_URL 决定）
          用于意图理解 / skill 路由
          上传内容：用户文本 + 文件路径 + 样本名等元数据
          不上传：.h5ad / .csv 等原始数据文件本身

原始数据文件
   │
   ▼
本地 omicsclaw 进程 → skill 脚本 → 本地输出
（不经过 Bot 平台，不经过 LLM 端点）
```

**诚实说明**：文字提示（包括你输入的 skill 名、参数、文件路径、样本编号）**会**按照 OpenAI 兼容协议上传到你在 `LLM_BASE_URL` 配置的 LLM 端点。如果你使用 OpenAI、DeepSeek 等商业服务，这些文本会离开你的机器。原始组学数据文件本身不会上传，但文件路径和样本名也是数据——若路径中含有受试者编号或病人 ID，请在运行前改名。

### 模式 3：远程执行（OmicsClaw-App + 远端服务器）

App UI 在本地运行，数据和计算在用户自己控制的远端服务器上执行，通过 SSH 隧道连接。

```text
本地 OmicsClaw-App（UI / Electron）
   │
   │  SSH 隧道（加密，仅 127.0.0.1 绑定）
   ▼
远端服务器（用户自控）
   oc desktop-server → skill 脚本 → workspace 目录
   原始数据、中间产物、报告全部留在远端磁盘

App 只拉取：图表 / report.md 等 artifact
原始数据不回传本地
```

远端服务器是用户自己拥有的机器，等同于本地处理——前提是用户确实 own 该服务器且无他人共享访问权限。需要用户保证：SSH 密钥安全存放、远端 `oc desktop-server` 仅绑 `127.0.0.1` 不暴露公网（控制面端点额外要求 `OMICSCLAW_REMOTE_AUTH_TOKEN` Bearer token）、服务器访问控制到位、多用户共享 GPU 服务器时设独立工作目录与文件权限隔离。

## 启用前 / 启用后（与云端工具对比）

| 维度 | 云端 SaaS 分析平台 | OmicsClaw |
|---|---|---|
| 原始数据落点 | 平台服务器 | 本地（或用户自托管远端） |
| 模型推理 | 平台 GPU | 本地或用户配置的 LLM |
| 第三方共享 | 取决于服务条款 | 由用户自己决定 |

## 用户操作清单（你需要做的）

- **用商业 LLM API 时**：确认其数据不留存条款（OpenAI / Anthropic / 国内厂商各有差异）
- **Bot 的 `.env` 不要 commit**：把 `.env` 加进 `.gitignore`，API key 与 bot token 均在其中
- **远端服务器的 SSH 密钥单独保管**：不要复用 root 密钥，推荐 `ed25519` 独立密钥对
- **报告分享前手动检查**：确认报告中不含病人 ID、受试者编号或真实姓名
- **多人共享 GPU 服务器时**：设独立工作目录并配置文件权限隔离（`chmod 700`）
- **文件路径 / 样本名也是数据**：路径含敏感词时考虑改名后再运行 Bot 模式
