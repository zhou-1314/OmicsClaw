# Domain docs

OmicsClaw uses one primary repository context with a Bench-specific supplement.

## Before exploring

1. Read `docs/CONTEXT.md` for authoritative project vocabulary.
2. Read ADRs under `docs/adr/` that touch the area being investigated.
3. For Bench work, also read `docs/bench/CONTEXT.md`.
4. Read `README.md`, root `SPEC.md`, and `AGENTS.md` as required by the
   repository maintenance contract.

If a referenced document does not exist, proceed without treating its absence
as a diagnosis by itself.

## Vocabulary and decisions

- Use terms as defined in the primary context instead of inventing synonyms.
- Treat the Bench context as a supplement, not a replacement for the primary
  context.
- Surface any conflict with an ADR explicitly. Do not silently override an
  accepted decision.
- If an important concept is missing from the glossary, record it as a design
  documentation gap.
