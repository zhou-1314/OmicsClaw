2.1 · 支点一

LLM 评审主席 · 只在两端介入

— in —

LLM 做什么

  • 挑选：根据 query + 数据特征 + param_hints，从 N 个候选方法挑 5 个 fan-out
  • 解读：N 个方法的 cross-method NMI + 复合分数 + consensus output → 带矛盾标注的 markdown 报告
— out —

LLM 不做什么

  • 不做 mode-voting / 加权平均 / Hungarian alignment
  • 不动 typed artifacts；不改写 cluster labels；不参与 BC ranking 的统计计算
  • 任何 LLM 调用对应到统计合并 → 视为越权 bug

统计合并交给类型化算子

kmodes · weighted majority · LCA-R (latent class) ——所有合并都在 runtime/consensus/operators/ 里，纯 Python / R 子进程， 确定性可复现。

这是 SACCELERATOR "人类专家选 BC" 思想的 LLM 落地——首次实现。 不变量 grep-tested 长期锁定。

OMICSCLAW · FALSIFIABILITY XII · HOW EACH CLAIM CAN FAIL

5 · Falsifiability

每条声明都有对应的 CI / grep 测试

"可证伪"的工程化版本——任何范式声明都要写出对应的 fail 条件，编进 CI 或 grep test。 这是 OmicsClaw 区别于"我们的方法更好（凭感觉）"类工作的核心姿态。

主张怎么证伪
consensus 比最好的单方法更稳 self-consistency: AMI stdev consensus > best member → fail
consensus 在 GT 比对上不输给单方法 DLPFC 151673 hero: 任一 hard metric consensus < best_member - noise_floor → fail
A/B 边界是硬的 报告里看不到 banner → 视为代码 bug
LLM 不做统计合并 grep 任何 LLM 调用对应到 mode-voting / 加权平均 → 视为越权 bug
evaluation chair 不能凭空打分 --llm-judge 模式下 α/β 调整必须 ≤ ±0.2 (ADR 0011)
interpretation never claims biology without evidence grep 'evidence.markers == []' in interpreted_assignments.json → bug (ADR 0012)
interpreted 不污染 typed namespace grep 'analysis://typed' interpreted_*.json 写入操作 → 越权
interpretation_faithfulness ≥ 1.00 verbatim recorded LLM 输出回归测试
marker_grounding_rate ≥ 0.60 on real LLM DeepSeek 实测 = 1.000 (Slide-seq hippocampus 2026-05-20)
DLPFC interpreted ARI ≥ 0.45 hero floor RUN_INTERPRET_DLPFC=1 gated CI
