## Step 3 — _render_r_enhanced() function (add as module-level function)

```python
def _render_r_enhanced(
    output_dir: Path,
    figure_data_dir: Path,
    r_enhanced: bool,
) -> list[str]:
    """Run R Enhanced rendering pass. Always called after Python figures are complete."""
    if not r_enhanced:
        return []
    from skills.singlecell._lib.viz.r import call_r_plot
    r_figures_dir = output_dir / "figures" / "r_enhanced"
    r_figures_dir.mkdir(parents=True, exist_ok=True)
    r_figure_paths: list[str] = []
    for renderer, filename in R_ENHANCED_PLOTS.items():
        out_path = r_figures_dir / filename
        call_r_plot(renderer, figure_data_dir, out_path)
        if out_path.exists():
            r_figure_paths.append(str(out_path))
    return r_figure_paths
```

## Step 4 — Call in main(), after write_result_json / write_standard_run_artifacts

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python -> R subprocess | call_r_plot() invokes Rscript via RScriptRunner; R output is a PNG file |
| figure_data_dir -> R | R reads CSVs from figure_data/; those CSVs were written by the Python skill |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | figure_data/ CSVs | accept | CSVs are written and read in the same output_dir; no user-controlled path injection in this wiring |
| T-18-02 | Denial of Service | R subprocess hang during --r-enhanced | mitigate | RScriptRunner has timeout=120 built in; call_r_plot() emits warning on RScriptTimeoutError, never hangs skill |
| T-18-03 | Repudiation | R_ENHANCED_PLOTS maps wrong renderer name | mitigate | registry.R dispatcher exits status 1 for unknown renderer, Python captures as warning — no silent data corruption |
| T-18-04 | Information Disclosure | r_enhanced_figures paths in result.json | accept | local paths only, no secrets; consistent with how python figure paths are already stored |
</threat_model>
